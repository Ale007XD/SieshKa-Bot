# Changelog

Все заметные изменения в проекте будут документированы в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/en/1.0.0/).

## [1.0.0] - 2024-02-05

### Added

#### Основной функционал
- Telegram бот для заказа еды
- Ролевая система (админ, менеджер, кухня, упаковщик, курьер)
- 3-х уровневая структура меню (категории → подкатегории → товары)
- Корзина с модификаторами товаров
- Полный жизненный цикл заказа (10 статусов)
- Система оплаты (наличные, карта, перевод)

#### Управление
- Панель администратора
- Управление категориями и товарами
- Управление персоналом
- Архивирование товаров и категорий
- Статистика и отчеты
- Импорт/экспорт меню

#### API
- REST API на FastAPI
- Аутентификация через JWT
- CRUD операции для меню и заказов
- Пагинация и фильтрация

#### Инфраструктура
- Docker и Docker Compose
- PostgreSQL база данных
- Redis для кэша и очередей
- Celery для фоновых задач
- Nginx reverse proxy
- Система бэкапов с отправкой в Telegram

#### Тестирование
- Unit тесты для сервисов
- Тесты для state machine
- Тесты API endpoints
- Интеграционные тесты

### Technical

#### Стек технологий
- Python 3.11+
- aiogram 3.x (Telegram Bot API)
- FastAPI (REST API)
- SQLAlchemy 2.0 (ORM)
- Alembic (миграции)
- Celery (фоновые задачи)
- PostgreSQL 15+
- Redis 7

#### Архитектура
- Clean Architecture
- Service Layer паттерн
- Repository паттерн
- State Machine для заказов
- Optimistic locking для конкурентности

### Security
- Валидация всех входных данных
- SQL-инъекции защищены через ORM
- JWT для API аутентификации
- Ролевая модель доступа (RBAC)
- Логирование действий администраторов
